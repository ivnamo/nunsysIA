# Prompt: Crear tests

Actua como Test Engineer.

Objetivo:

Crear tests para: `[COMPONENTE O CASO]`.

Antes de implementar, lee:

- `.cursor/rules/testing.mdc`
- docs relevantes del componente.

Requisitos:

- Usar pytest.
- Tests pequenos y deterministas.
- Mockear LLM cuando sea posible.
- No requerir llamadas pagadas para tests basicos.
- Cubrir caso success.
- Cubrir al menos un caso error o edge.
- Usar fixtures claras.

Entrega:

- Tests creados.
- Que comportamiento cubren.
- Como ejecutarlos.
- Dependencias o mocks usados.
